# Feature Demo Recorder — Implementation Plan

**Goal:** Create a Claude Code skill that records UI feature demos as GIFs via Playwright and maintains a features index document.

**Spec:** `docs/superpowers/specs/2026-03-23-feature-demo-recorder-design.md`

## Architecture

The skill is a single `SKILL.md` file that instructs Claude how to:
1. Parse "Demo" vs "Demo and Document" from the user's prompt
2. Ensure prerequisites (seed data, dev servers, output directories)
3. Use Playwright MCP tools to navigate the UI and record video
4. Convert the recording to a high-quality GIF with ffmpeg
5. Optionally update `docs/features/index.md` with the feature entry

No custom code files needed — the skill orchestrates existing tools (Playwright MCP, Bash/ffmpeg, Read/Write/Edit).

## File Structure

```
.claude/skills/feature-demo-recorder/
  SKILL.md                          # The skill definition
docs/features/
  index.md                          # Features index (created/maintained by skill)
  assets/                           # GIF output directory
```

## Dependencies

- Playwright MCP tools (browser_navigate, browser_click, browser_snapshot, browser_type, etc.)
- ffmpeg (installed at /opt/homebrew/bin/ffmpeg)
- Existing mise tasks (db:seed)

## Tasks

### Task 1: Create the skill file

**File:** `.claude/skills/feature-demo-recorder/SKILL.md`

Create the skill with YAML frontmatter and full process documentation.

The skill must instruct Claude to:

1. **Parse the prompt:**
   - Extract feature name from the user's message
   - Determine mode: "Demo" (GIF only) vs "Demo and Document" (GIF + index update)
   - Generate a kebab-case slug from the feature name

2. **Check prerequisites:**
   - Verify dev servers are running (curl localhost:5173 and localhost:8000)
   - If not running, tell the user to start them
   - Ensure `docs/features/assets/` directory exists
   - Check if DB is seeded by hitting the API; if not, run `mise run db:seed`

3. **Record the feature demo:**
   - Use Playwright MCP `browser_navigate` to open `http://localhost:5173`
   - Use Playwright MCP `browser_resize` to set viewport to 1920x1080
   - Start video recording via Playwright MCP tools
   - Navigate the UI based on the feature description — perform clicks, scrolls, waits as needed to demonstrate the feature
   - Add brief pauses between actions so the recording is readable
   - Stop recording when the demo is complete

4. **Convert to GIF:**
   - Run ffmpeg two-pass conversion on the recorded video:
     ```
     ffmpeg -y -i <video> -vf "fps=24,scale=1920:-1:flags=lanczos,split[s0][s1];[s0]palettegen=max_colors=256:stats_mode=full[p];[s1][p]paletteuse=dither=floyd_steinberg" docs/features/assets/<slug>.gif
     ```
   - Check file size — must be under 100MB (hard limit, no files over 100MB can be checked in)
   - If over 100MB, re-encode at lower resolution (step down: 1920→1280→800) or lower fps (24→15→10) until under 100MB
   - Report the final file size to the user
   - Clean up the source video file

5. **Update index (Document mode only):**
   - Read `docs/features/index.md` (or create with empty template if missing)
   - Check if a section for this feature already exists (match by heading)
   - If exists: update the description and GIF path
   - If new: append a new section with heading, AI-generated description, and GIF embed
   - Regenerate the ToC at the top of the file
   - The description should be 2-3 sentences summarizing what the feature does, written by Claude based on what it observed during the demo

**Checklist:**
