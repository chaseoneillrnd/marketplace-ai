# Receiving Code Review

## Core Principle

Verify before implementing. Ask before assuming. Technical correctness over social comfort.

## Process

1. Read completely
2. Restate requirements
3. Verify against actual codebase
4. Evaluate technical soundness
5. Respond or implement

## Rules

- No agreement theater ("You're absolutely right!", "Great point!")
- Clarify ALL unclear items before implementing any
- Verify external suggestions don't break functionality
- Push back with technical reasoning when appropriate

## When to Push Back

- Suggestion breaks functionality
- Reviewer lacks context
- Violates YAGNI
- Conflicts with team lead decisions

## Acknowledging Correct Feedback

Show understanding through action: "Fixed. [Description]" — not gratitude expressions.
